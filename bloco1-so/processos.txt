# PARTE C — Questões de síntese

# 1. O campo PPID em /proc/$$/status indica o processo pai do shell.
#    Execute ps -o pid,ppid,cmd e localize o par shell -> processo pai.
#    O que esse encadeamento sugere sobre como os processos sao
#    criados no Linux?

Resposta 1:
o ppid mostra quem iniciou o shell.
com isso, da para entender que no linux os processos vao criando outros processos, como uma cadeia de pai e filho


# 2. O arquivo /proc/<PID>/status mostra o campo State com uma letra
#    (S, R, Z...). Depois de executar kill $FILHO e rodar ps novamente,
#    o que aconteceu com o processo filho? O que isso te faz pensar  
#    sobre a vida de um processo?

Resposta 2:

depois do kill $filho, o processo filho terminou e sumiu do ps.
isso mostra que no linux os processos nascem, executam algo e depois morrem
